## ğŸ”’ Security Vulnerabilities

**DO NOT** open public issues for security vulnerabilities.

### Reporting

Email **matthew@vt-infinit.com** with:
- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### Response Timeline

- **24 hours**: Initial acknowledgment
- **72 hours**: Preliminary assessment
- **7 days**: Patch development (for critical issues)
- **14 days**: Public disclosure (coordinated)

### Responsible Disclosure

We practice **coordinated disclosure**:
1. Report received and acknowledged
2. Patch developed and tested
3. Fix deployed to production
4. Public advisory published
5. Reporter credited (if desired)

### Bug Bounties

We don't currently have a formal bug bounty program, but we deeply appreciate security researchers. Significant findings may receive:
- Public recognition (with permission)
- Exclusive achievement badges
- Priority feature requests
- Swag and merchandise
